# FinPulse — AI-ассистент для фондового рынка России

## Краткое описание

FinPulse — full-stack веб-приложение с AI-ассистентом для анализа новостей и данных российского фондового рынка.

Проект собирает, суммаризирует и интерпретирует финансовые новости, помогает пользователю быстрее понять, какие события могут быть важны для рынка РФ, отдельных компаний, секторов и инвестиционных решений.

Ключевая идея проекта — не просто пересказывать новости, а объяснять их возможное влияние на рынок, риски и инвестиционный контекст.

---

## Проблема, которую решает продукт

Инвесторы и частные пользователи сталкиваются с несколькими проблемами:

- информационный шум: большое количество новостей без понятной оценки важности;
- нехватка быстрой интерпретации событий именно в контексте российского фондового рынка;
- сложность самостоятельной оценки рисков, последствий и возможного влияния новости;
- необходимость быстро понимать, что сегодня важно для рынка, портфеля или отдельных бумаг.

Потребность пользователя можно сформулировать так:

> Быстро понять, что сегодня важно и чем это может закончиться для рынка или портфеля.

## Как работает продукт

High-level flow:

1. Система получает новости из заданных источников.
2. Backend обрабатывает данные и передаёт новости в AI/NLP-слой.
3. AI-слой выполняет:
   - суммаризацию новости;
   - выделение ключевых фактов;
   - классификацию;
   - оценку sentiment/impact;
   - генерацию аналитического комментария;
   - выделение рисков и неопределённостей.
4. Пользователь видит обработанные новости в ленте.
5. AI-чат отвечает на вопросы пользователя в контексте финансовой тематики и рынка РФ.
6. Профиль пользователя и настройки используются для персонализации сценариев.

## Архитектура MVP

Проект построен как full-stack MVP и включает несколько основных частей:

```text
FinPulse
├── frontend        # React + TypeScript интерфейс
├── backend         # FastAPI backend
├── PostgreSQL      # хранение пользователей, чатов и данных приложения
├── AI/NLP layer    # Ollama, GigaChat, rules-анализ
├── Docker Compose  # локальный запуск сервисов
└── docs            # документация проекта
```

Основные компоненты:

- Frontend отвечает за пользовательский интерфейс, страницы приложения, авторизацию, новости, чат и профиль.
- Backend предоставляет API, управляет бизнес-логикой, пользователями, новостями, AI-интеграциями и проверками состояния.
- PostgreSQL используется для хранения данных приложения.
- AI/NLP-слой отвечает за обработку новостей, суммаризацию, интерпретацию и ответы ассистента.
- Docker Compose позволяет запускать проект локально как набор связанных сервисов.

## Контейнеризация и деплой

Проект контейнеризирован через Docker Compose.

Локальная конфигурация поднимает основные сервисы приложения:

- frontend;
- backend;
- PostgreSQL;
- Ollama;
- reverse proxy для API-запросов.

Для production-сценария предусмотрен отдельный файл:

```text
docker-compose.prod.yml
```

Основной запуск MVP:

```bash
docker compose up -d --build
```

После запуска:

- Frontend: `http://localhost:3000`
- API через reverse proxy: `http://localhost:3000/api/*`
- Backend внутри Docker-сети: `http://backend:8000`
- Ollama API внутри Docker-сети: `http://ollama:11434`

## Ollama для AI-чата

Закрытый чат FinPulse использует локальную модель Ollama.

При запуске через Docker Compose сервис `ollama-init` проверяет и подтягивает модель из переменной окружения `OLLAMA_MODEL`.

Пример переменных окружения:

```env
OLLAMA_URL=http://ollama:11434
OLLAMA_MODEL=qwen2.5:1.5b
OLLAMA_MAX_CONCURRENCY=1
OLLAMA_CHAT_TIMEOUT_SEC=180
OLLAMA_CHAT_NUM_PREDICT=700
```

## GigaChat для публичной ленты

FinPulse может использовать GigaChat API как AI-слой для обработки RSS-новостей.

Модель получает заголовок, RSS-анонс и доступный текст статьи, после чего возвращает структурированный результат:

- summary;
- facts;
- conclusion;
- risks;
- indicator.

Если GigaChat недоступен или ответ невалиден, публичная лента автоматически использует быстрый rules-анализ.

Пример переменных окружения:

```env
NEWS_USE_LLM=true
NEWS_LLM_FOR_PUBLIC_RSS=true
GIGACHAT_AUTH_KEY=<Authorization key из личного кабинета>
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_MODEL=GigaChat
GIGACHAT_CHAT_MAX_TOKENS=700
GIGACHAT_NEWS_MAX_TOKENS=700
NEWS_PUBLIC_RSS_LLM_LIMIT=5
```
